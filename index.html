Last Odyssey官网｜Kingdom 模块设计 注册 60 万｜峰值 DAU 12 万｜满级玩家 3 万 展开详情

介绍与展示

`Kingdom` 是 Last Odyssey 官网中的预热型网页玩法：以游戏内美术资源为核心载体，将 Web3 用户熟悉的 farming 逻辑转译为“城堡养成”的可视化玩法， 用于承接 Launch 前流量、维持社区活跃，并为后续 Token listing 做用户教育与转化准备。

城堡养成主视图、资源进度、Chest、Quest 与外部社媒任务等内容。官网从信息展示升级为可参与的预热入口。

结果表现

  • 累计注册用户 `60 万+`
  • 峰值 `DAU 12 万`
  • 满级玩家 `3 万`

设计思路

  • 让用户在官网阶段建立代入感，通过升级、收集、任务与奖励形成上线前的连续参与动机。
  • 以任务链路承接社区传播和留存，同时把关键行为与后续 Launch/Token 节奏对齐。

我的工作

  • 负责玩法设计与数值设计：城堡成长、资源消耗、任务推进与奖励反馈的整体逻辑。
  • 设计任务结构与资源循环，并结合数据表现持续总结优化，提升理解、参与与转化。
Last Odyssey官网｜Hero 模块设计 40 万玩家参与 NFT 共创活动｜OBT 研发期填补运营空窗 展开详情

介绍与展示

`Hero` 模块是两次 OBT 之间研发阶段的官网互动活动，用于填补社区运营空窗期：通过“共创 + 投票 + NFT 产出 + OBT2 实装”的闭环， 把社区热度持续转化为可沉淀资产与可讨论内容。

多阶段流程（Create Hero / Voting / NFT Minting）与可展开说明，便于玩家理解规则、跟进进度并参与投票与领取。

结果表现

  • NFT 共创活动累计吸引 `40 万` 玩家参与。

活动目标

  • 用可投票、可共创的机制，把社区玩家从“关注者”转为“参与者”。
  • 把共创结果沉淀为 NFT 英雄资产，并在 OBT2 中实装，形成真实闭环。
  • 在官网侧建立可复用的多阶段活动模板（创建、投票、铸造与结果发布）。

核心机制

  • 社区玩家投票初选候选英雄。
  • 开发团队筛选出 5 个入围。
  • 全体玩家再次投票决出最终结果。
  • 制作成 NFT 英雄，并在 OBT2 实装。

我的工作

  • 整体方案设计：活动流程、页面结构、阶段规则、投票与结果发布逻辑。
  • 推动多阶段活动落地：从社区投票到开发筛选再到全员投票，保障规则清晰、节奏可控。
  • 与研发协作完成链路闭环：官网活动 → NFT 英雄生成 → OBT2 实装呈现。
Last Odyssey官网｜Badge 模块设计 SBT 徽章系统｜空投预期 展开详情

介绍与展示

`Badge` 模块用于构建 SBT（Soulbound Token）徽章体系，把玩家在官网各模块的关键行为沉淀为“不可转移的身份与成就资产”，用于活动承接与空投预期管理， 同时为 OBT 阶段的资格、奖励发放与玩家分层提供统一依据。

徽章中心按模块与等级展示成就进度，帮助玩家理解成长目标，并把“官网参与”转为可持续的身份积累与后续活动承接。

核心价值

  • 对玩家：把分散的玩法与任务串成可见的成长路径。
  • 对运营：形成可复用的资格门槛与奖励梯度底座。
  • 对产品：把官网行为数据转为结构化标签，服务后续转化。

系统设计要点

  • 徽章分层：按模块（如 Kingdom/Hero 等）与难度等级组织，形成清晰的成长路径。
  • 发放规则：围绕关键行为/里程碑定义达成条件，减少“纯签到”噪音指标。
  • 链路闭环：官网行为 → 资格/奖励 → OBT 体验反馈，保证玩家能感知“徽章价值”。

我的工作

  • 设计 SBT 徽章体系：徽章结构、等级规则、解锁条件与展示方式。
  • 打通官网玩法与 OBT 承接：将徽章作为资格/奖励规则的一部分，提升玩法联动。
  • 输出产品落地细节：信息层级、玩家引导、埋点维度与后续运营复用方式。
Last Odyssey官网｜Sanctuary 模块设计 OBT2 配套活动｜任务与排行｜合作方内容与奖励发放 展开详情

介绍与展示

`Sanctuary` 模块是 OBT2 的配套活动入口：把游戏内“圣所争夺/占领”的核心体验同步到官网， 通过任务、消耗、排行、充值等动作组织活动节奏，同时承接合作方内容曝光与奖励发放。

官网侧用地图形态展示圣所争夺与占领态势，并在左侧提供合作方内容入口；玩家可直接理解活动目标、参与路径与奖励机制。

合作方承接

  • 展示合作方内容位，提供活动联动入口与曝光资源。
  • 为玩家发放合作方奖励，补强激励与参与动机。

核心玩法与系统

  • 任务体系：把活动目标拆解为可执行任务（参与、协作、消耗、成长等），形成可复用模板。
  • 资源消耗与贡献：把玩家投入转化为贡献值，用于排行与奖励梯度。
  • 排行机制：支持个人/阵营（或公会）维度的竞争与结算，强化对抗与持续参与。
  • 充值/付费动作：与活动节奏结合，明确“付费能带来什么”，避免破坏公平感。

我的工作

  • 定义 OBT2 阶段目标与节奏：任务、消耗、排行与结算逻辑。
  • 设计官网侧“争夺态势展示”：把圣所占领情况做地图可视化呈现，增强参与与讨论。
  • 对接合作方联动：规划展示位与奖励发放链路，确保规则清晰、可运营、可复用。

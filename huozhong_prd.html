Last Odyssey｜火种重燃计划

0. 项目定位

火种重燃计划是 Last Odyssey 的赛季制文明竞争玩法。

玩家通过 NFT 质押获得火种，通过文明推演获取文明积分（CP），以城邦为组织参与地图竞争与排行榜竞争，并最终获得 USDC 奖励。

1.7 设计原则

原则

说明

赛季驱动原则

所有核心玩法必须绑定 Epoch 循环（创建→竞争→结算→重置），确保系统具备周期性重启能力

结构清晰原则

所有资产（CP / Embers / Time Marks / USDC / ODY）必须具备明确来源、用途与生命周期，避免语义混乱

低门槛参与原则

新用户在无历史积累情况下仍可进入核心玩法与竞争体系，不因历史数据产生参与壁垒

组织化优先原则

设计优先服务 City-State 结构，强化团队行为高于个人行为的价值分配

付费与玩法统一原则

所有付费行为必须直接对应玩法收益或效率提升（如 Simulation / NFT质押），避免纯金融化抽象消费

结果可解释原则

所有奖励（USDC / ODY）必须可追溯至行为贡献（CP / Embers / Simulation），保证结算透明

不确定性驱动原则

关键奖励机制（如圣杯）必须保留随机性与竞争性，增强参与动机与留存

数值防失控原则

通过 Epoch 重置与阶段清算控制数值规模，避免长期累积导致系统失衡或老玩家碾压

资产循环原则

NFT / Embers / Time Marks 必须参与至少一个闭环循环（获取→使用→消耗/重置）

体验统一原则

所有页面与模块需统一术语体系与交互逻辑，降低学习成本与认知负担

2. 功能模块

2.1. 城邦系统

2.1.1 创建城邦

玩家可通过两种方式参与赛季：

创建城邦

需创建者持有5枚火种，可创建城邦，城邦可最多容纳包括创建者在内共20名成员。

创建城邦不会消耗火种，仅要求持有数量。

城邦命名

创建城邦必须为城邦命名

字符长度：20

支持：英文 / 数字 / “-” / “_”

城邦名称必须本赛季唯一，往过赛季使用过的名字，在本赛季只有原使用者可以继续使用，新使用者不可使用。

屏蔽词见【文本-屏蔽词】

2.1.2 加入城邦

未持有赛季火种的玩家仍可参与玩法，但只能加入已有城邦。

城邦成员将根据 加入城邦的先后顺序 自动获得对应成员编号，该编号将作为命运圣杯结算的匹配依据。

排行榜加入

  • 所有已创建的城邦将统一展示在排行榜列表中。
  • 未加入任何城邦的玩家，可以浏览所有城邦信息，并向最多 1个城邦同时发起加入申请。
  • 玩家可随时撤销当前申请，并重新申请其他城邦。

邀请链接加入

  • 每个城邦支持专属邀请链接分享。
  • 通过链接进入的玩家，将直接打开该城邦的「加入申请弹窗」，可一键提交加入申请。

城主审批机制

  • 城主可在「成员管理页面」查看所有待审核申请，并进行以下操作：
  • 同意加入
  • 拒绝申请（被拒绝玩家本阶段无法再申请加入该城堡，但可以通过邀请加入该城堡）
  • 审批结果即时生效。

加入规则限制

  • 每个玩家 同一时间仅可申请加入1个城邦
  • 玩家在未加入城邦前，可自由撤销并重新申请
  • 加入成功后，自动从申请列表移除

玩家加入状态

  • 未加入城邦（可申请）
  • 已申请（待审批）
  • 已加入城邦
  • 申请被拒（可重新申请）

2.1.3 离开城堡

城邦成员退出与踢人机制

城邦成员可通过两种方式离开城邦：

  • 自主退出，任何成员可用
  • 城主移除，仅城主可用

移除原则

  • 资产归属：CP 与火种均与成员绑定，不属于城邦资产，城邦资产做对应减法。
  • 结算冻结：参与圣杯成员不可退出，每阶段阶段结算前最后24小时不可退出
  • 状态即时更新：退出行为实时生效，不回溯历史数据

可踢人/退出条件

  • 未处于「命运圣杯」已报名状态
  • 距离本阶段结束 ＞ 24小时

被移除成员的移除结果：

  • 立即退出城邦
  • 火种保留在个人账户，城邦火种数减少
  • CP 保留在个人账户，城邦CP减少
  • 不参与当前城邦后续结算

2.1.4 城邦资产

城邦CP=所有城邦内成员的CP总和

城邦火种数=所有城邦内成员的火种数总和

城邦的资产随人员变动实时变化

2.2 火种系统（Embers）

2.2.1 火种定义

火种是一种web2积分，仅在当前赛季有效，赛季结束后自动失效，不可跨赛季继承，新赛季需重新质押获得

火种可用于：

  • 创建城邦
  • 每周获取时间刻痕
  • 参与命运圣杯奖励

2.2.2 火种获取规则

火种仅可通过 NFT 质押获得，质押功能仅赛季期间开放。

同一NFT不可重复质押，质押期间NFT不可转移，到期后自动质押。

本赛季的NFT和过往赛季的NFT都可通过质押获得火种，但要求质押时长不同。

2.4.2 命运圣杯

基本机制

命运圣杯（Ember Tide）为周期性文明奖励事件，每个阶段（7天）开启一次结算。

系统每期开放 10个圣杯（Holy Grail），城主需在报名阶段选择其中 1个圣杯 参与争夺。

周期示意

状态

时间

可报名

Day1 00:00 - Day6 24:00

报名结束

Day7 00:00

正在随机

Day8 00:00

结果生成

Day8 00:00 - Day8 08:00

可领取

Day8 08:00

报名规则

  • 仅城主可操作报名
  • 城邦火种数量 ≥ 12
  • 每阶段仅可选择 1个圣杯
  • 阶段最后24小时禁止报名
  • 报名后城邦锁定进入对应圣杯竞争池

结算机制

结算时系统将依次随机确定：

  • 1个圣杯圣杯
  • 1个成员编号

仅同时满足以下条件的城邦进入竞争：

  • 报名同一圣杯且该圣杯被命中
  • 对应编号成员参与本次判定

胜负判定

满足条件的城邦按以下优先级排序决胜：

  1. 火种数量（Embers）
  2. 文明积分（CP）
  3. 达成时间（先达成优先）

优先级更高者获胜。

奖励分配

获胜城邦获得本期圣杯奖励，并按以下规则分配：

  • 仅城邦内火种持有者参与分配
  • 按火种持有数量比例分配奖励

2.5 城邦退出和防作弊机制

2.5.1 积分规则：

城邦内成员退出后，CP和火种对应减少，CP和火种绑定成员

2.5.2 退出条件

已报名命运圣杯的城邦，在本命运圣杯结算完成后，成员方可退出。

2.5.3 其他防BOT

玩家推出每阶段赛季结算前24小时，无法退出。

获取时间刻痕和文明推演必须在城邦内进行，未加入/创建城邦时功能不可用
